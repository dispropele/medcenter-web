# Тестирование МедЦентра

Полный набор тестов для проверки всех основных функций приложения медицинского центра.

## Установка

1. Установите зависимости:
```bash
npm install
```

## Запуск тестов

### Выполнить все тесты
```bash
npm test
```

### Запустить тесты в режиме наблюдения (watch mode)
```bash
npm run test:watch
```

### Получить отчет о покрытии кода
```bash
npm run test:coverage
```

## Структура тестов

Тесты находятся в папке `/tests/` и разделены по функциональным модулям:

### 1. `auth.test.js` - Тесты аутентификации
Проверяет:
- ✅ Отображение страницы входа
- ✅ Вход с неправильными учетными данными
- ✅ Успешный вход администратора
- ✅ Успешный вход пациента
- ✅ Регистрация нового пользователя
- ✅ Валидация данных при регистрации (имя, email, пароль)
- ✅ Требование аутентификации для защищенных страниц
- ✅ Логаут и уничтожение сессии

### 2. `appointments.test.js` - Тесты записей к врачу
Проверяет:
- ✅ Просмотр записей пациентом (только его записи)
- ✅ Просмотр всех записей администратором
- ✅ Получение списка специализаций и врачей
- ✅ Создание новой записи пациентом
- ✅ Создание записи администратором (от имени пациента)
- ✅ Получение деталей конкретной записи
- ✅ Изменение статуса записи
- ✅ Получение врачей по специализации (API)

### 3. `contracts.test.js` - Тесты договоров
Проверяет:
- ✅ Просмотр всех договоров (только для администратора)
- ✅ Получение списка пациентов для создания договора
- ✅ Создание нового договора
- ✅ Валидация обязательных полей
- ✅ Получение деталей договора с квитанциями
- ✅ Отказ в доступе для неадминистраторов

### 4. `receipts.test.js` - Тесты квитанций
Проверяет:
- ✅ Просмотр всех квитанций
- ✅ Получение списка договоров и услуг для создания квитанции
- ✅ Создание новой квитанции с услугами
- ✅ Валидация данных квитанции
- ✅ Получение деталей квитанции
- ✅ Изменение статуса квитанции (оплачено/ожидает)

### 5. `checks.test.js` - Тесты чеков об оплате
Проверяет:
- ✅ Просмотр всех чеков об оплате
- ✅ Создание нового чека об оплате
- ✅ Валидация обязательных полей
- ✅ Поддержка разных методов оплаты (наличные, карта, перевод)
- ✅ Автоматическое изменение статуса квитанции на "Оплачено"

### 6. `admin.test.js` - Тесты администрирования
Проверяет:

#### Специализации:
- ✅ Просмотр всех специализаций
- ✅ Добавление новой специализации
- ✅ Валидация имени специализации
- ✅ Удаление специализации

#### Услуги:
- ✅ Просмотр всех услуг
- ✅ Добавление новой услуги с описанием и ценой
- ✅ Валидация данных услуги
- ✅ Удаление услуги

#### Анализы:
- ✅ Просмотр анализов
- ✅ Добавление нового анализа
- ✅ Управление ценами анализов

#### Пользователи:
- ✅ Просмотр всех пользователей
- ✅ Проверка ролей пользователей

## Интеграционные тесты

Все тесты используют реальную базу данных (SQL.js) и реальное приложение Express.
Каждый тест полностью изолирован и не влияет на другие тесты.

## Что тестируется

- ✅ **Аутентификация и авторизация** — вход, регистрация, доступ по ролям
- ✅ **CRUD операции** — создание, чтение, обновление, удаление записей
- ✅ **Валидация данных** — проверка обязательных полей и формата
- ✅ **Бизнес-логика** — правильность расчета сумм, смены статусов
- ✅ **Безопасность** — ограничение доступа для неавторизованных пользователей
- ✅ **API endpoints** — все REST API работают корректно

## Устранение неполадок

### Если тесты зависают
Используйте флаг `--forceExit`:
```bash
npm test -- --forceExit
```

### Если тесты не находят модули
Убедитесь, что все зависимости установлены:
```bash
npm install
```

### Для отладки конкретного теста
```bash
npm test -- --testNamePattern="should login admin user"
```
