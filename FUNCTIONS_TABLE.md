# Таблица подпрограмм и операций системы управления медицинским центром

| Наименование подпрограммы | Описание подпрограммы | Входные переменные | Выходные переменные | Назначение |
|---|---|---|---|---|
| **Регистрация пользователя** | Форма для создания учетной записи нового пользователя в системе | name, login, password, role, dob, phone, address | user_id, success_message | Регистрация новых пользователей (пациентов, врачей, администраторов) в системе |
| Добавление/редактирование пользователя | Форма для добавления или изменения данных пользователя администратором | user_id, name, login, role, spec_id (для врачей) | success_message, error_message | Управление данными пользователей администратором |
| **Специализации** | | | | |
| Просмотр справочника специализаций | Список всех специализаций врачей с возможностью добавления, редактирования, удаления | — | specializations[] (id, name) | Просмотр и управление справочником специализаций |
| Добавление/редактирование специализации | Форма для добавления новой или изменения существующей специализации | name | spec_id, success_message | Добавление и редактирование данных о специализациях |
| Удаление специализации | Операция удаления специализации из справочника | spec_id | success_message, error_message | Удаление неиспользуемых специализаций |
| **Услуги** | | | | |
| Просмотр справочника услуг | Список всех услуг, оказываемых медицинским центром | — | services[] (id, name, price) | Просмотр и управление справочником услуг |
| Добавление/редактирование услуги | Форма для добавления новой или изменения существующей услуги | name, price | service_id, success_message | Добавление и редактирование услуг |
| Удаление услуги | Операция удаления услуги из справочника | service_id | success_message, error_message | Удаление неиспользуемых услуг |
| **Анализы** | | | | |
| Просмотр справочника анализов | Список всех видов анализов, доступных в центре | — | analyses[] (id, name) | Просмотр и управление справочником видов анализов |
| Добавление/редактирование анализа | Форма для добавления нового или изменения существующего вида анализа | name | analysis_id, success_message | Добавление и редактирование видов анализов |
| **Назначения (Приемы у врачей)** | | | | |
| Просмотр списка назначений | Таблица всех назначений с фильтрацией по датам и врачам | from_date, to_date, doctor_id | appointments[] (id, patient, doctor, datetime) | Просмотр всех приемов пациентов |
| Создание новой записи на прием | Форма для создания нового назначения пациента у врача | patient_id, doctor_id, datetime, reason | appointment_id, success_message, error_message | Создание нового приема пациента у врача |
| Проверка на пересечение времени | Проверка отсутствия конфликта времени приема у врача | doctor_id, datetime | is_available (true/false), conflict_message | Предотвращение двойного бронирования одного врача |
| Изменение назначения | Форма для редактирования данных существующего назначения | appointment_id, new_datetime, new_doctor_id | success_message, error_message | Изменение времени или врача для приема |
| Удаление назначения | Операция удаления назначения из системы | appointment_id | success_message, error_message | Удаление отмененных приемов |
| Печать направления к врачу | Вывод печатного документа-направления на основе назначения | appointment_id | pdf_document, formatted_data (patient_name, doctor_name, spec_name, datetime) | Формирование и печать направления к врачу |
| **Контракты (Договора с пациентами)** | | | | |
| Просмотр списка контрактов | Таблица всех контрактов с пациентами | — | contracts[] (id, patient, start_date, end_date) | Просмотр всех договоров о предоставлении услуг |
| Создание нового контракта | Форма для оформления договора с пациентом | patient_id, start_date, end_date | contract_id, success_message | Оформление договора на оказание услуг |
| Изменение контракта | Форма для редактирования данных контракта | contract_id, new_start_date, new_end_date | success_message, error_message | Изменение сроков действия договора |
| Удаление контракта | Операция удаления контракта | contract_id | success_message, error_message | Удаление завершенных или отмененных договоров |
| **Финансы - Квитанции (Receipts)** | | | | |
| Просмотр списка квитанций | Таблица всех квитанций по периодам | from_date, to_date | receipts[] (id, contract_id, date, patient) | Просмотр всех платежных квитанций |
| Создание новой квитанции | Форма для создания квитанции платежа | contract_id, receipt_date | receipt_id, success_message | Создание записи о платеже от пациента |
| Изменение квитанции | Форма для редактирования квитанции | receipt_id, new_date | success_message, error_message | Изменение данных платежа |
| Удаление квитанции | Операция удаления квитанции | receipt_id | success_message, error_message | Удаление ошибочных записей платежей |
| **Финансы - Чеки (Checks)** | | | | |
| Просмотр списка чеков | Таблица всех чеков-отчислений | — | checks[] (id, receipt_id, service_id, amount, date) | Просмотр всех чеков об оказанных услугах |
| Создание нового чека | Форма для добавления чека к квитанции | receipt_id, service_id, amount | check_id, success_message, error_message | Запись об оказанной услуге и ее стоимости |
| Проверка дублирования услуг | Проверка отсутствия одной услуги дважды в одной квитанции | receipt_id, service_id | is_duplicate (true/false), error_message | Предотвращение дублирования услуг в одной квитанции |
| Изменение чека | Форма для редактирования чека | check_id, new_service_id, new_amount | success_message, error_message | Изменение информации о услуге/стоимости |
| Удаление чека | Операция удаления чека | check_id | success_message, error_message | Удаление ошибочных записей об услугах |
| **Медицинская карта пациента** | | | | |
| Поиск и просмотр карты | Форма для поиска пациента и вывода его истории | patient_id или patient_name | patient_data, appointments[], analyses[], referrals[] | Просмотр полной медицинской истории пациента |
| **Отчеты** | | | | |
| Отчет по назначениям | Таблица приемов с группировкой по специализациям, за выбранный период | from_date, to_date, group_by | appointments_report[] (spec_name, doctor_count, total_visits) | Анализ нагрузки по приемам врачей по специализациям |
| Отчет по услугам | Таблица оказанных услуг с фильтрацией и группировкой | from_date, to_date, doctor_id, service_id, group_by, sort_by | services_report[] (doctor_name, service_name, patient_name, date) | Анализ оказанных услуг с различными фильтрами |
| Отчет по выданным направлениям | Таблица направлений с фильтром по услугам/анализам | service_id (опционально), group_by | referrals_report[] (service_name, count, dates) | Анализ выданных направлений на анализы |
| Отчет по загруженности и выручке врачей | Таблица врачей с кол-вом посещений и выручкой, с опциональной группировкой по специализациям | from_date (опционально), to_date (опционально), spec_id (фильтр), group_by | doctor_revenue_report[] (doctor_name, spec_name, visit_count, total_revenue) | Анализ эффективности врачей по количеству приемов и доходам |
| Печать отчета | Вывод отчета на печать через браузер | — | pdf_document / printer_output | Формирование печатного документа отчета |
| **Валидация и проверки** | | | | |
| Проверка даты начала и конца | Валидация корректности периода (начало ≤ конец) | from_date, to_date | is_valid (true/false), error_message | Предотвращение ошибок при выборе периода |
| Проверка пересечения приемов | Проверка отсутствия двойного бронирования врача | doctor_id, datetime, appointment_id (для редактирования) | is_available (true/false), conflict_message | Предотвращение двойных приемов одного врача |
| Проверка диапазона дат контракта | Валидация корректности дат начала и конца контракта | start_date, end_date | is_valid (true/false), error_message | Предотвращение ошибок при оформлении контракта |
| Проверка дублирования услуг в чеке | Валидация отсутствия одной услуги дважды в квитанции | receipt_id, service_id | is_duplicate (true/false), error_message | Предотвращение дублирования услуг |
| **Администрирование** | | | | |
| Просмотр пользователей | Таблица всех зарегистрированных пользователей в системе | role (фильтр, опционально) | users[] (id, name, login, role, phone) | Управление учетными записями пользователей |
| Блокировка/разблокировка пользователя | Операция блокирования доступа пользователя | user_id, status (active/blocked) | success_message | Контроль доступа пользователей к системе |

---

**Примечание:** Все подпрограммы содержат встроенные проверки валидации данных и выводят соответствующие сообщения об ошибках при неправильном вводе. Система использует дату-время в формате DD.MM.YYYY для хранения в базе данных и YYYY-MM-DD для отображения в HTML-формах.
